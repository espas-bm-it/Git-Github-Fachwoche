Arbeiten mit Git

Um mit Git über Windows Arbeiten zu können, ist die Installation von Git Bash von Nöten. Nach der erfolgreichen Installation kann das Arbeiten mit Git losgehen.

Über das Windows Terminal können verschiedenste Git Befehle ausgeführt werden, die unter anderem auch das Arbeiten mit GitHub ermöglichen. Neben der Arbeit mit dem Windows Terminal gibt es zwar die Desktop Version für GitHub. Diese ist zwar einfach zu bedienen, aber hat auch viel Fehlerpotential weswegen die meisten davon abraten. Die Bedienung über das Terminal kann zwar abschreckend wirken, aber am Ende ist es keine Zauberei. Um das Bedienen etwas angenehmer zu gestalten habe ich eine Liste der Wichtigsten Git- Befehle erstellt, diese ist weiter unten zu finden. Sobald man diese etwas verinnerlicht hat, ist man sehr wahrscheinlich sogar schneller als mit der GitHub Desktop Version und Vermeidet so auch unnötige Fehler die vom Programm aus entstehen können.

Auflistung der wichtigsten Befehle

Befehle Beschreibung
git add* Alle Aktualisierung wird in den nächsten Commit aufgenommen.
git comit –m "commit Beschreibung" Speicherung mit zusätzlicher Beschreibung.
git push Lädt die Datei im Repository hoch.
git push –set-upstream origin branch_name Vor dem Ersten Push, Push den ganzen Branch.
git merge Wird zum Zusammenfügen von z.B. Branches verwendet.
git status Zeigt den Status an.
git branch -a Zeigt die Branchhierarchie an.
git log Zeigt alle commits an die gemacht wurden.
git branch "Branchname" Erstellt einen neuen Branch mit dem gewählten Namen.
git log --oneline Zeigt die Hash-Codes der bisherigen Commits an.
git checkout Branchname/Commit Hash-Code Ermöglicht die Navigierung auf Branches oder auch Commits.
